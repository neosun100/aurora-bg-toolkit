Why customer-baseline hangs 30+ seconds

A short anatomy of the original problem.

T+0      Switchover triggered
T+10s    PREPARATION (RDS internal)        ← not optimisable
T+13s    bg plugin SuspendConnectRouting   ← DOWNTIME STARTS
T+17s    SuspendConnectRouting released    ← race with DNS update
         ┌── DNS already on Green IP  → instant recovery (~4s total)
         └── DNS still on stale Blue IP
              ├── customer baseline (no connectTimeout):
              │     TCP SYN hangs 30s, kernel retries
              │     → 35-57s total downtime
              └── v4 (connectTimeout=1000ms):
                    1s timeout, retry succeeds
                    → 5-7s total downtime
